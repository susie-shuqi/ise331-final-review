# Chapter 17 TLS/SSL 入门复习（中文）

> 目标：用适合初学者的方式复习 Chapter 17 中和 TLS/SSL 有关的重点。

## 1. 一句话理解 TLS/SSL

TLS（Transport Layer Security）可以理解为“给网络连接加一个安全保护罩”。当你访问 `https://` 网站时，浏览器和服务器会先协商安全参数，然后再传输真正的数据。

SSL（Secure Sockets Layer）是 TLS 的前身。日常说“SSL 证书”时，很多时候其实是在泛指现代 HTTPS 使用的 TLS 证书和 TLS 连接。

## 2. TLS 想解决什么问题？

TLS 主要提供三类安全能力：

1. **机密性（Confidentiality）**：别人即使抓到网络流量，也不应该看懂内容。
2. **完整性（Integrity）**：传输过程中数据不应该被偷偷修改。
3. **认证（Authentication）**：浏览器需要确认自己连接的服务器确实是目标网站，而不是攻击者伪装的假网站。

Chapter 17 的选择题第 10 题强调：TLS 同时提供数据完整性、认证和数据机密性。

## 3. 对称加密和非对称加密：TLS 为什么两种都用？

TLS 使用对称加密和非对称加密的组合。

### 对称加密

对称加密就像两个人共用同一把钥匙：

- 加密用这把钥匙。
- 解密也用这把钥匙。
- 优点：速度快，适合加密大量网页内容、图片、表单数据等。
- 问题：一开始双方怎么安全地拿到同一把钥匙？

### 非对称加密

非对称加密有一对钥匙：公钥和私钥。

- 公钥可以公开。
- 私钥必须保密。
- 它适合用来做身份认证、密钥交换等。
- 缺点：比对称加密慢，不适合直接加密大量网页内容。

### TLS 的基本思路

TLS 通常会先用非对称密码学相关机制安全地协商出会话密钥，然后用对称加密保护后续的大量数据。这样既能安全地建立连接，又能保持较高性能。

## 4. TLS Handshake（握手）是什么？

在真正传输网页内容之前，浏览器和服务器要先“握手”。这个过程可以理解为：

1. **打招呼**：浏览器告诉服务器自己支持哪些 TLS 版本、加密套件等。
2. **服务器证明身份**：服务器发送证书，让浏览器检查“你是不是我想访问的网站”。
3. **协商密钥**：双方协商后续通信要用的会话密钥。
4. **开始加密通信**：握手完成后，浏览器和服务器用协商好的密钥保护数据。

Chapter 17 的章节摘要明确指出：TLS 会话在被保护之前，需要先通过握手交换密码学信息和密钥。

## 5. HTTPS 和 TLS 的关系

HTTP 本身是明文协议。HTTPS 可以理解为：

```text
HTTPS = HTTP + TLS
```

也就是说，网页请求和响应仍然是 HTTP 的语义，但它们被包在 TLS 保护的连接里面传输。

## 6. 浏览器地址栏里的锁代表什么？

浏览器地址栏的锁通常表示当前网站使用 HTTPS/TLS。它主要说明：

- 连接经过加密。
- 浏览器已经检查服务器证书。
- 网络中间人不能轻易读取或篡改传输内容。

但要注意：锁并不等于“这个网站一定善良”。钓鱼网站也可能有 HTTPS。锁只能说明连接层面受到 TLS 保护，不保证网页内容本身可信。

## 7. SSL Stripping Attack（SSL 剥离攻击）

SSL stripping attack 的基本想法是：攻击者试图把用户原本应该使用的 HTTPS 连接降级成 HTTP。

一个初学者可以这样理解：

- 正常情况：用户访问银行网站，应使用 `https://bank.example`。
- 攻击情况：攻击者让用户停留在不安全的 `http://bank.example` 或伪造页面。
- 结果：用户以为自己在正常访问网站，但数据可能以明文方式暴露给攻击者。

常见防护思路包括：

- 用户主动确认网址是 `https://`。
- 网站使用 HSTS（HTTP Strict Transport Security）要求浏览器只通过 HTTPS 访问。
- 避免在公共 Wi-Fi 上忽视浏览器安全警告。

## 8. Cookie 的 Secure 标记和 HTTPS

Chapter 17 的选择题第 2 题提到，带有 `Secure` 标记的 cookie 只会通过 HTTPS 连接发送。

这很重要，因为 cookie 经常用来维持登录状态。如果登录 cookie 被通过普通 HTTP 明文发送，攻击者可能在网络中截获它，并冒充用户登录。

初学者记忆方式：

```text
Secure cookie → 只走 HTTPS → 降低被明文偷走的风险
```

## 9. 考试速记

- TLS 是现代 HTTPS 的核心安全协议。
- SSL 是 TLS 的前身；现实中“SSL”有时被口语化地用来指 TLS。
- TLS 使用对称加密和非对称密码学的组合。
- TLS 握手发生在安全会话正式开始之前。
- TLS 提供机密性、完整性和认证。
- HTTPS = HTTP over TLS。
- `Secure` cookie 只应通过 HTTPS 发送。
- SSL stripping attack 的目标是把 HTTPS 降级为 HTTP。

## 10. 自测题

### 题目 1
TLS 提供哪些安全功能？

A. 只有机密性  
B. 只有认证  
C. 机密性、完整性和认证  
D. 只负责压缩网页

**答案：C。**

### 题目 2
为什么 TLS 不直接用非对称加密来加密所有网页数据？

**参考答案：**非对称加密计算成本较高，速度较慢；TLS 更常见的做法是用非对称密码学相关机制协商密钥，然后用速度更快的对称加密保护大量数据。

### 题目 3
`Secure` cookie 的作用是什么？

**参考答案：**它要求浏览器只通过 HTTPS 连接发送该 cookie，从而降低 cookie 在明文 HTTP 中被截获的风险。

### 题目 4
SSL stripping attack 的核心危险是什么？

**参考答案：**攻击者试图把用户应使用的 HTTPS 安全连接降级成 HTTP 明文连接，使用户数据可能被监听或篡改。
